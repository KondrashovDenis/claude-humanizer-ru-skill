# Voice samples

Положи сюда 5–10 своих текстов: посты, переписку, статьи, заметки — что не жалко скормить модели.

## Форматы

`.txt` или `.md`. Имя файла любое, лучше с понятным префиксом:

```
voice-samples/
├── tg-post-2024-09.md
├── tg-post-2024-12.md
├── email-to-client.md
├── personal-chat.txt
└── article-about-X.md
```

## Что skill будет делать с ними

При работе с любым текстом сначала прочитает все файлы из этой папки и составит профиль:

- **Длина:** среднее число слов в предложении и предложений в абзаце
- **Лексика:** любимые слова, обороты, тики, нелюбимые штампы
- **Пунктуация:** тире vs дефис, точки vs многоточия, восклицательные знаки, заглавные после знаков препинания
- **Структура:** как ты обычно начинаешь абзацы, как переходишь между мыслями, как заканчиваешь
- **Уровень формальности:** сленг, мат, жаргон, англицизмы
- **Личностные маркеры:** самоирония, скобки с уточнениями, тире с пояснениями, и т.д.

Потом перепишет твой целевой текст в этой манере, а не в обобщённо-«человеческой».

## Чем больше образцов — тем лучше

Минимум 3, оптимум 5–10. Слишком много (50+) тоже плохо: профиль получится размытым.

Если режимы письма у тебя разные (формальный для писем + разговорный для ТГ), можно сделать подпапки:

```
voice-samples/
├── formal/
│   └── ...
└── casual/
    └── ...
```

И в команде для skill сказать: «прогони как личную переписку, voice — casual».

## Приватность

Файлы из этой папки **не пушатся в репозиторий** — `.gitignore` режет всё, кроме этого README. Если ты форкнул репо и не хочешь чтобы образцы случайно попали в коммит — проверь, что `.gitignore` на месте.
