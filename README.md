# claude-humanizer-ru-skill

Skill для Claude Code: чистит русские тексты от типичных AI-штампов и опционально подгоняет под голос автора по образцам.

Адаптация [blader/humanizer](https://github.com/blader/humanizer) под русский.

## Что делает

Прогоняет текст через каталог из ~30 русских AI-паттернов в 11 категориях (канцелярит, кальки с английского, раздутая значимость, чатбот-артефакты, hedging, структурные штампы, типографика) и переписывает с учётом контекста.

### Контекстные режимы

- `tg-post` — пост в Telegram / соцсети (+ типографическая деградация)
- `letter` — деловое письмо (профи-типографику сохраняем)
- `personal` — личная переписка (+ типографическая деградация, сохраняем авторский бардак)
- `copywriting` — текст для сайта
- `audit` — анализ чужого текста на признаки AI-генерации

### Что добавлено сверх оригинала

Оригинал [blader/humanizer](https://github.com/blader/humanizer) заточен под английский, половина английских паттернов (em-dash overuse, tapestry, copula avoidance) на русский не переносится или работает иначе. В русской версии:

- **Каталог паттернов переписан с нуля** под русский — канцелярит, кальки, раздутая значимость, чатбот-артефакты, промо-лексика
- **Контекстные режимы** — допустимые штампы и агрессивность правки зависят от формата (пост ТГ, письмо, переписка, копирайтинг)
- **Типографическая деградация** для неформальных режимов — `—` → `-`, `…` → `...`, «ёлочки» → прямые кавычки, неразрывные пробелы → обычные. В деловом письме и копирайтинге профи-символы остаются — там они маркер грамотного автора, а не AI
- **Правило «не чини авторский бардак»** — опечатки, лишние запятые, смайлы `)` не исправляем. AI слишком грамотен, идеальная пунктуация сама по себе маркер
- **Режим аудита** — оценка 0–100% AI + топ паттернов с цитатами + вердикт (человек / гибрид / AI с правкой / чистый AI)

### Не использовать для

КП, договоры, ТЗ, техдока — там структура и формальность важнее «человечности», и попытка «оживить» сделает только хуже.

### Примеры работы

Сравнение «до / после» с разбором паттернов и самоаудитом — в [examples/before-after.md](examples/before-after.md). Сейчас там пример прогона типового AI-поста про продуктивность через режим `tg-post` (15 паттернов → 0).

## Использование

В Claude Code просто скажи что нужно — skill подхватится автоматически:

- «причеши этот текст»
- «убери AI-стиль»
- «прогони через humanizer»
- «проверь, не сгенерён ли этот текст AI»

Опционально укажи режим: «причеши как пост в ТГ», «как письмо клиенту», «это личная переписка».

## Voice calibration (опционально)

Положи 5–10 своих текстов в `voice-samples/` — посты, переписку, статьи, что не жалко. Skill сначала профилирует твой голос (длину фраз, любимые обороты, пунктуационные привычки), потом перепишет в твоей манере, а не в обобщённо-«человеческой».

Подробности — в `voice-samples/README.md`.

Файлы из этой папки в репозиторий не пушатся (`.gitignore` режет всё, кроме README).

## Кредиты

Адаптация [blader/humanizer](https://github.com/blader/humanizer) (MIT).

Оригинал: 4-фазная методика (паттерны → переписка → калибровка под голос → самоаудит) и общая структура skill'а.

В русской версии: каталог паттернов переписан под русский (канцелярит, англицизмы, кальки, специфика чатбот-артефактов в RU), добавлены контекстные режимы и режим аудита.
